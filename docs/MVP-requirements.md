# 家庭数字记忆盒

## 项目需求文档：家庭数字记忆盒（Memory Box）

- 版本：MVP v0.1
- 目标用户：有 0–6 岁孩子的家长（尤其是妈妈），希望轻松记录孩子成长瞬间
- 核心理念：用 AI 将杂乱的照片变成有温度的成长日记，让记忆可读、可感、可分享

## 一、项目目标（Why）

- 解决痛点：家长手机里有成千上万张孩子照片，但缺乏整理动力，珍贵瞬间被淹没
- 创造价值：自动将照片转化为“可读的成长故事”，降低记录门槛，增强情感连接
- 个人意义：帮助开发者（你）保持技术手感，探索 AI + 生活场景的结合，积累作品
- MVP 原则：不做相册管理、不做用户系统、不做复杂编辑——只聚焦“一张图 → 一段温暖文字”

## **二、目标用户画像**

| **用户** | **需求** | **场景** |
| --- | --- | --- |
| **你（开发者妈妈）** | 想记录孩子成长，但没时间手动写日记 | 孩子午睡时，随手上传一张照片，5秒获得一段文字 |
| **其他妈妈用户** | 想给家人分享孩子近况，但不想发一堆图 | 一键复制 AI 生成的文案，直接发到家庭群 |
| **祖辈（爷爷奶奶）** | 想了解孙辈日常，但看不懂零散照片 | 收到带文字描述的链接，一看就懂 |

## **三、MVP 功能清单（V1.0）**

| **功能模块** | **功能描述** | **优先级** |
| --- | --- | --- |
| **1. 照片上传** | 用户点击“添加闪光时刻”按钮，从设备选择一张 JPG/PNG 照片 | 🔴 高 |
| **2. AI 智能解读** | 调用 GPT-4 Vision API，分析照片内容，生成 30–60 字的成长日记 | 🔴 高 |
| **3. 结果展示** | 左侧显示原图，右侧显示 AI 生成的文字（带柔和背景+手写字体） | 🔴 高 |
| **4. 一键复制** | “复制文案”按钮，点击后将文字复制到剪贴板 | 🟡 中 |
| **5. 分享链接** | （可选）生成一个临时预览链接（如 `memorybox.vercel.app/share/abc123`），可发给家人 | 🟢 低（MVP 可省略） |

> ⚠️ 明确不做（V1 不包含）：
> 
> - 用户注册/登录
> - 多照片时间线
> - 视频支持
> - PDF 导出 / 打印
> - 付费功能

## **四、AI 提示词（Prompt）规范**

**角色设定**：温暖、细腻、有育儿经验的儿童成长记录员

**输出要求**：

- 字数：30–60 字
- 语气：亲切、真实、有爱，避免过度煽情
- 内容：基于图中可见信息（不编造），包含时间（若无则用“最近”）、行为、情绪
- 结尾：加一个贴切的 emoji（如 🌈, 🧸, 🌞, 🎨）

## **五、技术栈建议（轻量 + 快速上线）**

| **层级** | **推荐方案** | **理由** |
| --- | --- | --- |
| **前端框架** | Next.js (App Router) + Tailwind CSS | 你熟悉 React，Vercel 一键部署，支持 Server Actions |
| **AI 能力** | OpenAI GPT-4 Turbo with Vision (`gpt-4-turbo-2024-04-09`) | 支持图像理解 + 文本生成，API 稳定 |
| **部署平台** | Vercel | 免费、自动 HTTPS、全球 CDN，适合静态+Serverless |
| **存储（MVP）** | 无持久化（每次上传临时处理） | 避免用户系统，简化架构 |
| **可选增强** | Web Share API（移动端一键分享） | 提升分享体验 |

**开发提示**：用 Vercel AI SDK 可 10 行代码调用 Vision 模型
